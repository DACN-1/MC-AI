# VLA Action Head Requirements — NVIDIA RTX 5090 / Blackwell (sm_120) path.
#
# This is the *additive* sibling of requirements.txt. The cluster (RTX A5000,
# Ada/Ampere) keeps using requirements.txt unchanged — torch 2.4 + the pinned
# FlashAttention-2 wheel. Blackwell (sm_120) has no kernel image in those
# wheels, so the rented vast.ai 5090 box uses *this* file instead.
#
# Two deliberate differences from requirements.txt:
#   1. torch/torchvision/torchaudio are NOT listed here — they cannot resolve
#      from normal PyPI for Blackwell. Install them FIRST from the cu128 index:
#        pip install torch==2.7.* torchvision torchaudio \
#            --index-url https://download.pytorch.org/whl/cu128
#      (torch 2.7.0 is the first stable release shipping cu128 / sm_120 wheels.)
#      scripts/setup_vastai_5090.sh does this for you, then `pip install -r` here.
#   2. flash-attn is intentionally OMITTED. Stock flash-attn has no sm_120
#      kernel ("no kernel image available"), and there is no official prebuilt
#      wheel for cu128 + torch2.7 + cp312. VLAAgent already falls back to sdpa
#      attention when flash_attn is absent — numerically equivalent modulo fp16
#      reduction order. Do not `pip install flash-attn` on this box.
#
# Compute dtype: VLAAgent defaults to bf16 on Blackwell (sm_120), fp16 on
# Ampere/Ada — see VLAAgent.py compute_dtype selection. bf16's wider exponent
# keeps the no-FA2 sdpa softmax stable at larger batches, so the 5090 path
# pushes batch ~32 instead of the old fp16 batch-24 limit. Storage is still
# unconditionally fp16 (feature_cache.py:324), so caches stay shape-mergeable
# across A5000 fp16 and 5090 bf16 builds — per-feature drift is ~3rd-decimal
# noise vs an A5000 fp16 build and not material for the downstream BC head.
# Override either default with R1VA_LLAVA_DTYPE={fp16,bf16}.
#
# Python 3.12 (cp312) is required — same as requirements.txt — because numpy<2
# and the rest of the stack were validated there. Use a 3.12 venv or a 3.12
# base image (e.g. pytorch/pytorch:2.7.0-cuda12.8-cudnn9-runtime).
